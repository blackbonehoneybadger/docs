# Запуск токена $DOFFA со своего компьютера (для новичка)

Эта инструкция проведёт тебя от нуля до выпуска токена в **devnet** (тестовая
сеть Solana — бесплатно и безопасно). Когда всё получится в devnet, теми же
командами сделаем mainnet.

> На любом шаге, если что-то пошло не так — скопируй **красный текст ошибки**
> и пришли мне. Не угадывай, не удаляй ничего — просто покажи мне.

---

## Шаг 0. Что понадобится
- Компьютер с **Windows** или **macOS** и интернетом.
- 20–30 минут.
- Никаких денег: devnet бесплатный.

---

## Шаг 1. Установить Node.js
1. Открой сайт **https://nodejs.org**
2. Скачай версию с пометкой **LTS** (большая зелёная кнопка).
3. Установи (везде «Next/Далее», ничего менять не надо).
4. Проверь, что установилось. Открой терминал:
   - **Windows:** меню Пуск → набери **PowerShell** → открой.
   - **macOS:** Cmd+Space → набери **Terminal** → открой.
5. В терминале набери и нажми Enter:
   ```bash
   node -v
   ```
   Должна появиться версия, например `v20.x.x`. Если появилась — отлично.

---

## Шаг 2. Скачать проект с GitHub
Самый простой способ — скачать ZIP:
1. Открой репозиторий: **https://github.com/blackbonehoneybadger/docs**
2. Зелёная кнопка **Code** → **Download ZIP**.
3. Распакуй архив (например, на Рабочий стол). Появится папка `docs`.
4. Внутри найди папку `doffa/token` — с ней и работаем.

---

## Шаг 3. Открыть терминал в папке `doffa/token`
- **Windows:** открой папку `doffa\token` в Проводнике → в адресной строке
  сверху набери `powershell` и нажми Enter (откроется терминал уже в этой папке).
- **macOS:** открой Terminal и перетащи папку `token` в окно после команды `cd `:
  ```bash
  cd 
  ```
  (поставь пробел после `cd`, перетащи папку, нажми Enter).

Проверь, что ты в нужном месте:
```bash
ls
```
(на Windows — `dir`). Ты должен увидеть файлы `package.json`, `README.md`, папку `src`.

---

## Шаг 4. Установить зависимости
```bash
npm install
```
Подожди 1–2 минуты. Это скачает библиотеки Solana. Предупреждения (warning) — это нормально.

---

## Шаг 5. Создать конфиг
- **Windows:**
  ```bash
  copy .env.example .env
  ```
- **macOS:**
  ```bash
  cp .env.example .env
  ```
Для devnet-теста менять в нём ничего не нужно (там уже `CLUSTER=devnet`,
`DOFFA`, `100000000`).

---

## Шаг 6. Создать ключ владельца
```bash
npm run keygen
```
Создастся файл `owner.json` — это **твой главный ключ**.
🔒 Никому его не показывай (мне тоже), не выкладывай в интернет.

---

## Шаг 7. Получить бесплатные тест-SOL (на комиссии)
```bash
npm run airdrop
```
Должно написать баланс `2 SOL`.
Если выдаст ошибку про лимит — открой **https://faucet.solana.com**, выбери
**devnet**, вставь свой адрес (его печатает шаг 6 и шаг verify) и получи SOL там.

---

## Шаг 8. Выпустить токен
```bash
npm run create
```
Команда создаст токен `$DOFFA`, метаданные и заминтит 100 000 000 тебе.
В конце напечатает **адрес токена** и **ссылку на Solscan** — открой её,
увидишь свой токен вживую. 🎉

---

## Шаг 9. Проверить
```bash
npm run verify
```
Покажет объём, decimals и кто имеет права.

---

## Шаг 10. Попробовать сжигание (имитация «продали чашку»)
```bash
npm run burn -- 15
```
Сожжёт 15 токенов. Снова запусти `npm run verify` — объём уменьшится. Это и есть наша механика.

---

## Шаг 11. (по желанию для теста) Отозвать права
```bash
npm run revoke
```
После этого нельзя допечатать/заморозить токены. На devnet можно потренироваться;
на mainnet это делаем осознанно и в самом конце.

---

## Что дальше
- Получилось в devnet → переходим к **mainnet**: в `.env` меняем `CLUSTER=mainnet-beta`,
  кладём на кошелёк реальные SOL, готовим **постоянный логотип** (Arweave/IPFS) для `METADATA_URI`,
  и повторяем `create → verify → revoke`.
- Я помогу с логотипом-метаданными и подключу реальный адрес токена к сайту (страницы «Купить» и дашборд).

> Двигайся по одному шагу. После каждого можно написать мне «сделал, всё ок» или прислать ошибку — подскажу.
